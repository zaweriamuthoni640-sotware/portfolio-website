# KBC User Provisioning Project

## Description
This project demonstrates the process of user provisioning at KBC. 
It includes creating, managing, and deactivating user accounts 
to ensure secure and authorized access to systems.

## Purpose
To provide a clear, step-by-step method for granting access 
to staff based on their roles and responsibilities, maintaining security, 
and documenting all actions for accountability.

## How to Use
1. Obtain authorization from HR or the department supervisor.
2. Create user accounts with proper roles and permissions.
3. Assign temporary passwords and require first-time login change.
4. Document all actions, including usernames, systems accessed, and approval.
5. Disable or delete accounts when staff leave or change roles.
